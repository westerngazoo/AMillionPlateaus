# R-0070 — "Before this, study…" (missing prerequisites + guided plan)

- **Status:** Accepted
- **Milestone:** POC — Knowledge content
- **Owner:** Gustavo Delgadillo
- **Created:** 2026-07-16
- **Depends on:** R-0065 (the numbered curriculum paths that define learning ORDER), R-0063 (lesson
  progress) + R-0030 (mastery) — the "studied" signal, R-0069/R-0023 (the resources pinned per topic
  that the plan is built from), R-0056 (the NotebookLM/Gemini hand-off).
- **Realized by:** direct implementation — a pure `prereqs.js` (missing-prereq selector + plan-prompt
  builder) + a `#detail-prereqs` block on the open topic, driven by main.js.
- **Source:** the owner: "I wanna be able to study prerequisites if needed — can the companion guide
  me through, depending on what we're studying, a book, a YT course or such." Confirmed via
  AskUserQuestion: guidance by **hand-off to NotebookLM/Gemini**, prereqs built **from pinned resources**.

## 1. Statement

On an open topic that's a step in a curriculum path, a **"Before this, study…"** block lists the
**earlier steps you haven't studied yet** — the missing prerequisites — as numbered, tappable chips
(each flies to and opens that topic). A **"Guide me through them →"** hand-off copies a study-plan
prompt — the target topic, the ordered missing prereqs, and the **resources pinned on each** (your
YouTube videos, the seeded Khan/OpenStax/books) — and opens NotebookLM/Gemini so the model builds a
focused plan around what you're actually studying from. The block is hidden when the topic is the
first step, isn't in a path, or every earlier step is done.

## 2. Rationale

The graph encodes order (the numbered paths, R-0065) and mastery/lesson progress, but nothing said
"you're missing X before this." The learner had to notice the gap themselves. This closes it: the
prereqs are exactly the earlier path steps not yet studied, and — because resources are now pinned per
topic (R-0069/R-0023) — the guided plan is built from the learner's own sources, matching the
"depending on what we're studying, a book, a YT course" ask. Deriving prereqs from **path order** (not
bridge direction) is deliberate: `suggest-path.js` treats bridges as undirected and the seed spine
sets their direction inconsistently, so order is the only reliable prerequisite signal.

## 3. Acceptance criteria

- **AC1 — Missing prereqs, in order.** For a topic in a curriculum path, list the earlier steps whose
  id is neither mastered (R-0030) nor lesson-done (R-0063), in path order, each labelled with its
  1-based step number. Hidden when the topic is step 1, is in no path, or all earlier steps are done.
- **AC2 — Tappable.** Each prereq is a chip that flies to and opens that topic (reusing `flyTo` +
  `openPlateau`), so the block re-renders for the topic you jump to.
- **AC3 — Guided plan from pinned resources.** A "Guide me through them" hand-off (NotebookLM / Gemini
  / AI Studio) copies a prompt with the target, the ordered missing prereqs, and the resources pinned
  on each (title + URL; a "suggest one" note where none is pinned), then opens the tool — nothing sent
  automatically.
- **AC4 — Recomputed live + no bridge leak.** Recomputed on every topic open (reflecting fresh
  progress); hidden in the bridge/connection view; a `[hidden]` guard so it never lingers.
- **AC5 — Pure + additive + tested + model-free at the edge.** `prereqs.js` (`missingPrereqs` +
  `prereqPlanPrompt`) is pure/deterministic with `node --test`; `apps/web` only; no core/Rust/wasm
  change; no new dependency; the plan is generated by the hand-off (no in-app model dependency).

## 4. Constraints & non-goals

- **Order, not bridges** — prereqs come from the authored path sequence; bridge direction is not used
  (unreliable/undirected in the current data).
- **"Studied" = mastered OR lesson-done** — the same union the lens-path panel uses; either signal
  clears a prereq.
- **Non-goals (follow-ups):** capping/collapsing a very long prereq list (rare — normal in-order study
  keeps it short); an in-app (non-hand-off) plan; prereqs across DIFFERENT paths (only the topic's own
  path is considered — the first path it appears in); marking a prereq "studied" from this block
  (open it and use its Teach-me / mastery gate).

## 5. Decision log

| Date | Decision | Rationale |
|------|----------|-----------|
| 2026-07-16 | Derive prerequisites from path ORDER, not bridge direction | Bridges are undirected in suggest-path and inconsistently directed in the seed data; the numbered paths are authored in true learning order. |
| 2026-07-16 | Build the plan from the resources pinned on each prereq | Owner's choice; makes the guidance about what they're actually studying from (their channel / books), and reuses R-0069/R-0023 with no new machinery. |
