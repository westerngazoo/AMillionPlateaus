# API CONTRACTS — A Million Plateaus

## 1. mp-wasm JavaScript API

Realized by SPEC-0003 (R-0003). Generated by `wasm-bindgen` from the `mp-wasm`
crate; build with `wasm-pack build crates/mp-wasm --target web` → `pkg/`. The
`.d.ts` below is the authoritative shape (see `crates/mp-wasm/pkg/mp_wasm.d.ts`).

```typescript
import init, { WasmGraph } from "./pkg/mp_wasm.js";
await init();                       // instantiate the wasm module first

class WasmGraph {
    constructor();

    /** Add a plateau; returns the engine-generated plateau id (UUID string).
     *  e1/e2/e3 are the Grade-1 knowledge-space coordinates
     *  (Formal/Mathematical, Physical/Empirical, Creative/Expressive). */
    add_plateau(name: string, domain_id: string,
                e1: number, e2: number, e3: number): string;

    /** Connect two existing plateaus. The bridge rotor/grade are derived in
     *  mp-graph (Bridge::between) — never supplied by JS. Throws if either
     *  endpoint is unknown or a UUID is malformed. */
    add_bridge(from_id: string, to_id: string, concept: string): void;

    /** A plateau by id, or null for an unknown id. */
    plateau(id: string): PlateauDto | null;

    /** Is a single plateau reachable for the given reputation? Throws on a
     *  malformed reputation JSON or an unknown plateau (never a silent false). */
    is_reachable(plateau_id: string, wizard_rep_json: string): boolean;

    /** The set of plateau ids reachable for the given reputation
     *  (membership; order not significant). */
    reachable_plateaus(wizard_rep_json: string): string[];

    free(): void;                   // wasm-bindgen-managed; also Symbol.dispose
}

interface PlateauDto {
    id:          string;
    name:        string;
    description: string;
    position:    { e1: number; e2: number; e3: number };  // Grade-1 coeffs
}
```

### `wizard_rep_json` shape

Reputation crosses the boundary as a JSON **string** (a garust `Multivector`
has no serde derive). Each array is a coefficient vector in blade order
`[1, e1, e2, e12, e3, e13, e23, e123]`:

```json
{
  "domain_reps": { "<domain-uuid>": [0, 0.9, 0.1, 0, 0, 0, 0, 0] },
  "synthesis":   [0, 0, 0, 0, 0, 0, 0, 0]
}
```

Both fields are optional (default: empty map / all-zero synthesis). A
**scalar-only** reputation (arrays carrying only index 0) yields an empty
`reachable_plateaus` — the Sybil/fog property is preserved client-side.

> **Not yet exposed (deferred):** `save()`/`load()` byte snapshots (redb is
> native-only; belongs with the CRDT phase) and the `ReputationEngine` mutators
> (`record_traversal`/`propagate`) — reputation is supplied as JSON for now.

## 2. Alebrije Service REST API

```
POST /alebrije/ask
Content-Type: application/json

Request:
{
    plateau_id:           string,
    wizard_traversals:    [{ plateau_name: string, depth: number }],
    alebrije_name:        string,
    alebrije_components:  string[],        // ["Owl", "Octopus"]
    user_message:         string,
    conversation_history: [{ role: "user"|"assistant", content: string }]
}

Response 200:
{
    reply:            string,              // alebrije voice response, max 256 tokens
    suggested_plateaus: string[]           // 0-3 nearby plateau names hinted at
}

Response 429: rate limited
Response 500: Claude API error
```

## 5. CRDT Sync Protocol (Gun.js relay)

```
Node key structure:
  plateaus/{plateau_id}    → PlateauNode JSON
  bridges/{bridge_id}      → Bridge JSON
  resources/{resource_id}  → Resource JSON
  votes/{resource_id}/{wizard_id} → number (grow-only)

Sync:
  Client subscribes to plateaus/* → receives all plateau updates
  Client puts plateau → relayed to all connected peers
  Automerge changes serialized as base64, stored at:
    crdt/changes/{change_hash} → base64 encoded Automerge change
```
